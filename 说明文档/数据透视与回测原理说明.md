> 本文档面向实习生，结合项目代码 `fct_backtest.py`，从数据格式转换出发，拆解整个回测流水线的每一步原理。

## 一、原始数据：长表格式（Long Format）

从 `price_data.parquet` 读取后，`price_data` 是一个 DataFrame，大致结构如下：

| trade_date | code      | close_adj | is_st | trade_days | fct_1 | fct_2  | ... |
| ---------- | --------- | --------- | ----- | ---------- | ----- | ------ | --- |
| 2021-01-04 | 000001.SZ | 15.20     | False | 1825       | 0.032 | -0.015 | ... |
| 2021-01-04 | 000002.SZ | 8.75      | False | 2200       | 0.018 | -0.022 | ... |
| 2021-01-04 | 000003.SZ | 22.30     | True  | 300        | 0.005 | -0.008 | ... |
| 2021-01-05 | 000001.SZ | 15.45     | False | 1826       | 0.028 | -0.019 | ... |
| 2021-01-05 | 000002.SZ | 8.90      | False | 2201       | 0.021 | -0.025 | ... |

**特点**：每一行 = 一只股票 × 一个交易日的一条记录。所有信息（价格、因子值、股票状态）全堆在纵向行里。

**为什么长表不方便回测？**

- 要算"每只股票明天的涨跌幅"→ 得按股票分组，再按日期排序，再对每只股票逐行 `.shift()`，写起来是双重循环
- 因子值和收益不在同一个粒度上，要反复 `merge`，性能差、容易错位

## 二、透视：长表 → 宽表（Wide Format）

### 对应的代码行

```python
# fct_backtest.py 第 45 行
price_aligned = pd.pivot_table(
    price_df['close_adj'].reset_index(),
    index='trade_date',
    columns='code',
    values='close_adj'
)
```

原本数据是多级行索引，即索引以二元元组的形式存在，哪怕表格 **只有`close_adj`这 1 列价格** ，也能通过双层索引精准锁定「某一天某只股票」的收盘价，我们可以在读取 parquet 时就设定多级索引，也可以手动设置索引

```Python
# 读取文件时，直接指定两列作为行索引
df = pd.read_parquet("price_data.parquet", index_col=["trade_date", "code"])
```

```Python
# 原本trade_date、code是普通列，执行这句后转为双层行索引
df = df.set_index(["trade_date", "code"])
```

`price_df['close_adj']`：取出单列收盘价，**行索引还是双层 (日期 + 股票)**`.reset_index()`：把**两层行索引全部拆解，变成普通的数据列,而`pd.pivot_table`的`index`和`columns`参数 **只能接收普通列名，不能直接读取行索引** ，所以必须先解索引。**

注意：**`.reset_index()` 不只是专门解除多级索引** ，它是 **把任意层级的行索引（单层 / 多级），全部降级为普通数据列** ，同时给表格重新生成默认 0、1、2、3…… 的数字行索引。

### 参数含义

| 参数        | 值               | 作用                                 |
| ----------- | ---------------- | ------------------------------------ |
| `index`   | `'trade_date'` | 行维度：每一行代表一个交易日         |
| `columns` | `'code'`       | 列维度：每一列代表一只股票           |
| `values`  | `'close_adj'`  | 格子里的值：该股票在该日的复权收盘价 |

### 转换结果示意

**转换前（长表）**：

```
trade_date    code        close_adj
2021-01-04    000001.SZ   15.20
2021-01-04    000002.SZ   8.75
2021-01-05    000001.SZ   15.45
2021-01-05    000002.SZ   8.90
```

**转换后（宽表 `price_aligned`）**：

```
trade_date    000001.SZ   000002.SZ   ...
2021-01-04    15.20       8.75        ...
2021-01-05    15.45       8.90        ...
```

行是时间、列是个股、格子是价格——一个标准的 **trade_date × code 二维矩阵**。

## 三、为什么回测必须转成二维宽表

### 3.1 批量平移算未来价格，一行代码搞定全市场

```python
# fct_backtest.py 第 48-50 行
future_prices = price_aligned.shift(-period-1)   # 全市场未来 N 日价格
current_prices = price_aligned.shift(-1)          # 全市场次日价格
future_returns = (future_prices / current_prices - 1)  # 全市场未来 N 日收益率
```

对二维矩阵调用 `.shift(-N)`，等于把所有股票的列**统一向下平移 N 行**：

```
原始 price_aligned:
            000001.SZ  000002.SZ
2021-01-04  15.20      8.75
2021-01-05  15.45      8.90
2021-01-06  15.10      9.20

price_aligned.shift(-1):  ← 整体上移一行
            000001.SZ  000002.SZ
2021-01-04  15.45      8.90      ← 原来 01-05 的值
2021-01-05  15.10      9.20      ← 原来 01-06 的值
2021-01-06  NaN        NaN

未来收益率 = shift(-1) / 原值 - 1 → 每一只股票、每一天的次日收益率，一次矩阵除法全部算完
```

**如果在长表里做同样的事**：需要 for 循环遍历每只股票 → 分组 → 排序 → 计算 → 合并，代码量至少 10 倍，速度慢几十倍。

### 3.2 "每隔 period 天取一行"，对齐持有期

```python
# fct_backtest.py 第 54 行
future_returns = future_returns.iloc[::period].dropna(axis=0, how='all')
```

当持有期是 5 天时，`iloc[::5]` 每隔 5 行取一行。这样算出的收益率就自然对应"持有 5 天的收益"（避免了每天调仓的收益率和 5 天持有期不匹配的问题）。

二维宽表做这个切片操作就是一行代码；长表里要先考虑"每只股票每隔 N 天"的复杂逻辑。

### 3.3 因子与收益对齐：透视 → 堆叠 → 合并

因子分析的核心就是把**因子值**和**未来收益率**按（日期, 股票）匹配起来：

```python
# fct_backtest.py 第 56-60 行
returns_series = future_returns.stack()  # 宽表 → 长表（一维 Series）
returns_series.name = f'return_{period}d'
merged_df = pd.merge(
    factor_df,
    returns_series,
    left_on=['trade_date', 'code'],
    right_index=True
)
```

这里 `.stack()` 把宽表又"压"回长格式（宽→长的逆操作），目的就是和因子数据（天然是长格式的 `trade_date + code + factor_value`）做精确匹配。

**整个流程**：长表 → 透视成宽表（做矩阵运算）→ 部分压回长表（做 join 匹配）

## 五、关键概念速查表

| 概念           | 一句话解释                                  | 代码位置                |
| -------------- | ------------------------------------------- | ----------------------- |
| 长表 (Long)    | 每行一条记录，日期+股票+值全纵向堆叠        | 原始 parquet 数据       |
| 宽表 (Wide)    | 行=日期、列=股票、格=值，二维矩阵           | 第 45 行`pivot_table` |
| `.shift(-N)` | 所有股票列整体向上平移 N 行，拿到未来价格   | 第 48-49 行             |
| `.stack()`   | 宽表压回长表，用于 merge                    | 第 56 行                |
| IC             | 因子值与未来收益的 Spearman 秩相关系数      | 第 85 行                |
| ICIR           | IC 均值 / IC 标准差，衡量稳定性             | 第 91 行                |
| Q1~Q10         | 按因子值从小到大分 10 组，Q1 最小、Q10 最大 | 第 66-68 行             |
| 多空收益       | Q10 累计收益 - Q1 累计收益                  | 第 82 行                |
| MAD 去极值     | 中位数 ± 5.2×MAD 之外的极端值被截断       | 第 24-29 行             |
| Z-score 标准化 | 因子值转为均值 0、标准差 1                  | 第 31-34 行             |

## 六、一句话总结

**原始数据是"一条一条竖着堆"的长表 → 透视成"日期行 × 股票列"的棋盘矩阵 → 整张矩阵用 `.shift()` 一次算出全市场未来收益 → 再和因子值对齐算出 IC / 分层收益。** 整个回测的核心就是把数据在长表和宽表之间灵活切换，用宽表做批量矩阵运算，用长表做精确的 join 匹配。
