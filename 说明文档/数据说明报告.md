# 回测框架数据说明报告

## 一、数据文件概览

本回测框架使用两份 Parquet 格式的数据文件：

| 文件                   | 大小    | 行数      | 列数 | 时间范围                |
| ---------------------- | ------- | --------- | ---- | ----------------------- |
| `price_data.parquet` | ~179 MB | 5,766,680 | 13   | 2021-01-04 ~ 2025-11-28 |
| `fct_df.parquet`     | ~57 MB  | 5,663,561 | 2    | 2021-01-04 ~ 2025-10-31 |

---

## 二、price_data.parquet —— 行情数据

### 2.1 文件结构

- **格式**: Parquet（列式存储二进制格式）
- **索引**: 两层 MultiIndex — `[trade_date, code]`
  - `trade_date`: `datetime64[ns]`，交易日日期
  - `code`: `str`，股票代码（如 `000001.SZ`、`688981.SH`）
- **覆盖面**: 1187 个交易日 × 5344 只股票

### 2.2 字段说明

| 字段           | 类型        | 说明                                               |
| -------------- | ----------- | -------------------------------------------------- |
| `is_st`      | `bool`    | 是否为 ST 股票（`True`=是 ST，`False`=正常）   |
| `trade_days` | `float16` | 该股票截至当日的累计上市交易天数                   |
| `open`       | `float64` | 当日开盘价（不复权）                               |
| `high`       | `float64` | 当日最高价（不复权）                               |
| `low`        | `float64` | 当日最低价（不复权）                               |
| `close`      | `float64` | 当日收盘价（不复权）                               |
| `vol`        | `float64` | 当日成交量（股）                                   |
| `amount`     | `float64` | 当日成交额（元）                                   |
| `pct_chg`    | `float64` | 当日涨跌幅（%，如 -3.83 表示下跌 3.83%）           |
| `open_adj`   | `float64` | 后复权开盘价                                       |
| `high_adj`   | `float64` | 后复权最高价                                       |
| `low_adj`    | `float64` | 后复权最低价                                       |
| `close_adj`  | `float64` | **后复权收盘价**（回测计算收益率的核心字段） |

### 2.3 关键统计特征

- **数据完整性**: 所有 13 个字段零空值，数据完整
- **ST 股票占比**: 约 3%（176,707 行），在预处理阶段被过滤
- **上市不足一年**: 约 340,282 行（`trade_days < 365`），在预处理阶段被过滤
- **股票代码格式**: 深圳 `.SZ`（如 `000001.SZ`），上海 `.SH`（如 `600000.SH`），科创板 `689xxx.SH`
- **后复权价格范围**: `close_adj` 约 0.33 ~ 87,492，跨度极大（反映不同股价和复权累积效应）

### 2.4 复权价格的作用

框架计算未来收益率时使用的是 `close_adj`（后复权收盘价），这是量化回测的标准做法：

```
未来收益率 = 未来复权价 / 当前复权价 - 1
```

后复权价格已修正了分红送股对价格的影响，能真实反映持有期内的实际收益。

## 三、fct_df.parquet —— 因子数据

### 3.1 文件结构

- **格式**: Parquet（列式存储二进制格式）
- **索引**: 两层 MultiIndex — `[trade_date, code]，即多级行索引，索引以二元元组的形式存在`（与 price_data 结构一致）
- **覆盖面**: 1167 个交易日 × 5338 只股票
- **与 price_data 的关系**: fct_df 的索引是 price_data 索引的**真子集**（566 万 vs 577 万行），多出的 10 万行在 price_data 中（主要是 2025 年 11 月的数据，fct_df 只到 2025-10-31）

### 3.2 字段说明

| 字段      | 类型        | 说明                                         |
| --------- | ----------- | -------------------------------------------- |
| `fct_1` | `float64` | 因子 1，已标准化处理，均值≈0，标准差≈0.074 |
| `fct_2` | `float64` | 因子 2，原始值，均值≈0.38，标准差≈0.18     |

### 3.3 因子统计特征

| 统计量             | fct_1     | fct_2     |
| ------------------ | --------- | --------- |
| 有效值数量         | 5,636,878 | 5,626,211 |
| 空值数量           | 26,683    | 37,350    |
| 均值               | 0.0023    | 0.3797    |
| 标准差             | 0.0737    | 0.1771    |
| 最小值             | -0.908    | 0.030     |
| 25% 分位           | -0.034    | 0.260     |
| 50% 分位（中位数） | -0.002    | 0.340     |
| 75% 分位           | 0.031     | 0.454     |
| 最大值             | 1.492     | 3.126     |

**解读**:

- **fct_1** 近似以 0 为中心对称分布，已经做过一定程度的标准化，值域约 [-0.9, 1.5]
- **fct_2** 全部为正值（min=0.03），分布偏右，中位数约 0.34，可能是某种比率类因子（如换手率、波动率等）
- 两个因子都存在少量空值（约占 0.5%~0.7%），代码中通过 `fillna(0)` 做了填充处理

## 四、回测框架如何使用这两份数据

### 4.1 整体数据流

```
price_data.parquet  ──┐
                      ├── pd.concat(axis=1, join='inner') ──→ 合并数据
fct_df.parquet      ──┘
                              │
                              ▼
                   极端值处理（MAD 缩尾）
                              │
                              ▼
                   Z-Score 标准化 + 空值填0
                              │
                              ▼
                  剔除 ST 股票 + 上市不足1年
                              │
                              ▼
                     单因子分层回测
                              │
                              ▼
                   绩效分析 + 可视化 + Excel 输出
```

### 4.2 代码逐步解读

#### 步骤 1: 读取数据并合并

```python
price_data = pd.read_parquet('price_data.parquet')
fct_df = pd.read_parquet('fct_df.parquet')
price_data = pd.concat([price_data, fct_df], axis=1, join='inner')
```

- `axis=1`：水平拼接（列方向），把因子列追加到行情表右侧
- `join='inner'`：取交集，只保留两个 DataFrame 共有的 `(trade_date, code)` 行
- 合并后：15 列（13 个行情字段 + 2 个因子），约 566 万行

#### 步骤 2: 因子预处理

```python
price_data[factor_names] = extreme_MAD(price_data[factor_names])   # MAD 缩尾
price_data[factor_names] = standardize_z(price_data[factor_names].fillna(0))  # Z-Score 标准化
```

- **MAD 缩尾（extreme_MAD）**: 使用绝对中位差法，以中位数 ± 5.2×MAD 为上下界，将极端值截断。这比传统的均值±3σ 方法更稳健，不受极端值本身的影响
- **Z-Score 标准化（standardize_z）**: `(x - mean) / std`，使因子变为均值 0、标准差 1 的分布
- **fillna(0)**: 空值填充为 0（即标准化后的均值水平）

#### 步骤 3: 样本池过滤

```python
mask = (~df['is_st']) & (df['trade_days'] >= 365)
```

- 剔除 ST 股票（`is_st == True`）
- 剔除上市不足 365 个交易日的次新股
- 这两个条件同时满足才保留

#### 步骤 4: 因子分层回测（factor_analysis）

核心逻辑：

1. **构建收益率矩阵**: 将 `close_adj` 透视成 `trade_date × code` 的二维表
2. **计算未来收益率**: 对于每个持有期 `period`（1/5/10/20 天）:
   ```
   未来收益率 = close_adj[t+period] / close_adj[t] - 1
   ```
3. **因子-收益对齐**: 将因子值与对应持有期的未来收益率按 `(trade_date, code)` 合并
4. **每日分层**: 在每个交易日内，按因子值将股票分成 10 组（Q1~Q10），使用 `pd.qcut`（等数量分位）

#### 步骤 5: 绩效分析（analyze_factor_performance）

对每个持有期计算：

| 指标                       | 计算方式                                     | 含义                                               |
| -------------------------- | -------------------------------------------- | -------------------------------------------------- |
| **分位数累计收益**   | 各组日均收益 → 累乘                         | Q1（最低因子组）到 Q10（最高因子组）各自的累计净值 |
| **多空收益**         | Q10 累计收益 − Q1 累计收益                  | 做多最高组、做空最低组的收益差                     |
| **IC（信息系数）**   | 每日因子值与收益的 Spearman 秩相关系数的均值 | 因子预测能力的核心指标，绝对值越大越好             |
| **ICIR（信息比率）** | IC 均值 / IC 标准差                          | IC 的稳定性指标，越大越稳定                        |

#### 步骤 6: 因子方向判断（determine_factor_direction）

```python
if long_short_return > 0:
    direction = '正向因子'  # Q10 跑赢 Q1，因子值越大收益越高
else:
    direction = '负向因子'  # Q1 跑赢 Q10，因子值越小收益越高
```

#### 步骤 7: 组合绩效指标（calculate_portfolio_metrics）

对目标分位组（正向因子选 Q10，负向因子选 Q1）计算：

| 指标       | 公式                                | 说明                       |
| ---------- | ----------------------------------- | -------------------------- |
| 累计收益率 | `∏(1+r) − 1`                    | 整个回测期的总收益         |
| 年化收益率 | `(1+R)^(252/n) − 1`              | 折算到年化的收益率         |
| 最大回撤   | `max(1 − 当日净值/历史最高净值)` | 最大净值回落幅度           |
| 夏普比率   | `年化收益 / 年化波动率`           | 单位风险的超额收益         |
| 索提诺比率 | `年化收益 / 下行波动率`           | 只惩罚下行风险的夏普改进版 |
| 卡玛比率   | `年化收益 / 最大回撤`             | 收益与最大回撤的比值       |

### 4.3 当前代码正在回测什么

```python
factor_df = price_data['fct_2']  # 第 380 行：正在回测 fct_2
results = factor_analysis(factor_df, price_data, periods=(1, 5, 10, 20), quantiles=10)
```

当前脚本对 **fct_2** 做单因子回测，分别看 1天、5天、10天、20天四个持有期的表现，每期分 10 组。

### 4.4 输出产物

- **Excel 文件**（`单因子回测统计结果.xlsx`）：三个 Sheet
  - `汇总统计`: 各持有期的 IC、ICIR、累计/年化收益、最大回撤、夏普比率
  - `分位数收益`: Q1~Q10 各组的累计收益
  - `IC时间序列`: 每日 IC 值的时序数据
- **图表**: 分层累计收益曲线图 + 因子绩效仪表盘

## 六、Parquet 格式在本框架中的优势

1. **列式存储**: 回测只需要 `close_adj` + 当前因子列时，Parquet 不加载其他列，节省 IO 和内存
2. **内置类型信息**: 日期、字符串代码、浮点数值的类型固化，不会像 CSV 出现格式错乱
3. **高压缩比**: 两个文件共约 236MB，同样数据存 CSV 预估 1~3GB
4. **MultiIndex 原生支持**: 索引结构直接保存在文件中，读取即可用，无需额外 `set_index`
